## 项目概述

### 网站定位

个人网站是您在数字世界的 "数字名片"，是展示个人品牌、专业技能和作品的核心平台。根据您的需求，我们将打造一个现代化、功能完整的个人展示空间。

### 核心目标

* **展示专业能力**：突出您的技能、经验和项目成果

* **建立个人品牌**：塑造专业形象，提升行业影响力

* **拓展网络机会**：吸引潜在客户、雇主或合作伙伴

* **内容分享平台**：分享知识、见解和行业观点

### 目标受众

* 潜在雇主和招聘经理

* 行业同行和专业人士

* 潜在客户和合作伙伴

* 对您专业领域感兴趣的访客

## 技术栈选择

基于 2025 年最新技术趋势和最佳实践，推荐以下技术栈：

### 前端技术栈

#### React 生态

* **Next.js 14+**：React 全栈框架，支持 SSR/SSG/ISR

* **TypeScript**：类型安全，提升代码质量

* **TailwindCSS**：原子化 CSS，快速构建响应式界面

* **Shadcn/ui**：高度可定制的组件库

#### 辅助工具

* **Zod**：数据校验库

* **TanStack Query**：服务器状态管理

* **Zustand**：轻量状态管理

* **React Email**：邮件模板

* **Lucide React**：图标库

### 后端技术栈

#### 轻量级方案（推荐个人网站）

##### 方案 A：Serverless 架构

* **Cloudflare Workers**：边缘计算，免费额度充足

* **Supabase**：开源 Firebase 替代品，提供数据库、认证、存储

* **Prisma**：现代化 ORM，类型安全

##### 方案 B：传统后端

* **Spring Boot 3.x**：Java 生态，企业级稳定性

* **MySQL/PostgreSQL**：关系型数据库

* **Redis**：缓存和会话管理

### 数据库选择

| 数据库类型   | 推荐方案          | 适用场景   | 优势           |
| ------- | ------------- | ------ | ------------ |
| **关系型** | PostgreSQL    | 复杂数据关系 | 功能全面，开源免费    |
| **文档型** | Supabase      | 快速开发   | 集成度高，BaaS 服务 |
| **轻量级** | Cloudflare D1 | 边缘应用   | 低延迟，5GB 免费存储 |

## 页面结构细化

### 信息架构（Sitemap）

* 首页（/）

* 关于（/about）

* 项目（/projects，详情：/projects/[slug]）

* 博客（/blog，文章：/blog/[slug]）

* 服务（可选）（/services）

* 联系（/contact）

### 路由结构（Next.js App Router）

```text
app/
├─ layout.tsx
├─ page.tsx                     # 首页
├─ about/
│  └─ page.tsx
├─ projects/
│  ├─ page.tsx                  # 列表与筛选
│  └─ [slug]/
│     └─ page.tsx               # 详情页
├─ blog/
│  ├─ page.tsx
│  └─ [slug]/
│     └─ page.tsx
├─ services/
│  └─ page.tsx
└─ contact/
   └─ page.tsx
```

### 页面布局与关键模块

#### 首页

* 导航栏、Hero（价值主张与 CTA）、技能概览、精选项目、最新文章、品牌背书（徽标/证书）、页脚

#### 关于

* 个人简介、时间线（教育/经历）、技能矩阵（按领域与熟练度）、价值观与方法论、下载简历按钮

#### 项目

* 顶部筛选（按标签/技术栈/年份）、项目卡片网格、分页或懒加载、项目详情包含：背景、目标、角色与职责、技术栈、关键挑战与解决方案、成果指标（数据/图表）、链接（GitHub/预览）

#### 博客

* 文章列表（可按标签/主题筛选）、文章页：标题、作者与日期、阅读时长、目录（滚动高亮）、正文（代码高亮）、相关文章推荐、分享与订阅

#### 服务（可选）

* 服务内容、流程（发现→方案→交付→复盘）、定价卡片、FAQ、案例与见证（证言）

#### 联系

* 表单（姓名、邮箱、主题、内容，含校验与防机器人）、成功反馈与备用联系方式（邮箱、LinkedIn、GitHub）、地图（可选）

### 组件分层与目录约定

```text
components/
├─ ui/            # 原子级组件（Button、Input、Badge、Card ...）
├─ layout/        # 布局组件（Navbar、Footer、Sidebar ...）
├─ sections/      # 业务区块（HeroSection、ProjectsGrid ...）
└─ composite/     # 组合组件（ArticleWithTOC、ProjectDetails ...）

lib/              # 工具函数、hooks、常量
content/          # MD/MDX 文章与项目描述（可用 Contentlayer 或自建）
public/           # 静态资源（图像、图标）
```

### 内容模型定义（建议）

* 项目（Project）：`title`、`slug`、`summary`、`stack[]`、`tags[]`、`role`、`duration`、`highlights[]`、`metrics{}`、`links{ demo, repo }`、`images[]`

* 文章（Post）：`title`、`slug`、`description`、`date`、`readingTime`、`tags[]`、`coverImage`、`content`

### SEO 与可用性

* 每页设置唯一 `title` / `description` / `canonical`；文章页使用结构化数据（JSON-LD）

* 目录与跳转使用锚点与平滑滚动，支持键盘导航与焦点可见性

## 配色细化

### 品牌色与系统色

#### 主色（Primary — 专业蓝）

* 基准色：#165DFF

* 色阶（参考）：50 #EAF1FF，100 #D7E3FF，200 #A9C3FF，300 #7DA5FF，400 #4F86FF，500 #2B6CFF，600 #165DFF，700 #134ED4，800 #113FA9，900 #0E317F

#### 辅助色（Accent）

* 青色（提升科技感）：#06B6D4

* 紫色（强化高级感，备选）：#7C3AED

#### 中性色（Neutral Gray）

* 50 #FAFAFA，100 #F4F4F5，200 #E5E7EB，300 #D4D4D8，400 #A1A1AA，500 #71717A，600 #52525B，700 #3F3F46，800 #27272A，900 #18181B

#### 语义色（Semantic）

* 成功：#22C55E；警告：#F59E0B；错误：#EF4444；信息：#3B82F6

### 明暗模式配色映射

* 亮色模式：背景 `#FFFFFF`，文本 `#0F172A`，主按钮背景 `#165DFF`，主按钮文本 `#FFFFFF`

* 暗色模式：背景 `#0B1220`，文本 `#E5E7EB`，主按钮背景 `#3B82F6`（在暗色下略微降低饱和度以提高清晰度），主按钮文本 `#FFFFFF`

* 辅助元素（分隔线/卡片底色）：亮色 `#F8FAFC`，暗色 `#111827`

### 可访问性与对比度

* 正文文本对比度达到 WCAG 2.1 AA（≥ 4.5:1），重要标题力争 AAA（≥ 7:1）

* 主按钮与背景对比度 ≥ 4.5:1；禁用态采用降低饱和度与降低透明度，不改变语义色

* 链接在暗色模式下使用更亮的蓝色（如 `#3B82F6`）并增加下划线以强化可感知性

  --color-primary-600: #165DFF;
  --color-primary-700: #134ED4;
  --color-primary-800: #113FA9;
  --color-primary-900: #0E317F;

  --color-accent: #06B6D4;
  --color-success: #22C55E;
  --color-warning: #F59E0B;

  --color-info: #3B82F6;

  --color-bg: #FFFFFF;

### 使用建议

* 主色用于品牌元素、主按钮、链接与强调标题；辅色用于二级按钮、徽章与装饰性图形

* 在图表、代码高亮与信息提示中使用语义色，保持一致性

* 保持页面内最多 1 主色 + 1-2 辅助色，避免视觉噪声

## 项目文档与文档结构

### 文档目标与原则

* 面向维护与协作：让任何参与者在 10 分钟内理解项目现状与开发规范

* 及时更新：需求、设计、实现、部署与变更记录同步更新，避免文档与代码脱节

* 结构清晰：主题化分目录，单文件不宜超过 1000 行，必要时拆分

* 规范统一：遵循 Markdown 规范，避免 MD040/MD032/MD012 等常见错误；公式使用 LaTeX 格式

### 文档目录结构（建议）

```text
docs/
├─ overview.md            # 项目概览与目标、术语表
├─ roadmap.md             # 路线图与里程碑
├─ architecture/
│  ├─ frontend.md         # 前端架构、目录、状态管理、路由
│  ├─ backend.md          # 后端/Serverless 架构、API 约定
│  └─ data.md             # 数据模型、存储、缓存策略
├─ design/
│  ├─ colors.md           # 配色与可访问性
│  ├─ typography.md       # 字体与排版系统
│  └─ components.md       # 组件设计规范与示例
├─ content/
│  ├─ writing-guide.md    # 写作指南与标签规范
│  └─ publishing.md       # 发布流程、审校与归档
├─ seo/
│  ├─ keywords.md         # 关键词库与布局策略
│  └─ checklist.md        # 每页 SEO 检查清单
├─ ops/
│  ├─ deployment.md       # 部署方案与环境变量管理
│  ├─ monitoring.md       # 监控与告警
│  └─ security.md         # 安全与合规
└─ changelog.md           # 版本变更记录（遵循 Keep a Changelog）
```

### 关键文档清单

* 项目概览（overview.md）：愿景、范围、目标、非目标、术语表

* 架构文档（frontend/backend/data）：技术决策与约束、目录结构、交互协议、异常处理策略

* 设计规范（colors/typography/components）：设计 token、组件状态、交互反馈

* 内容规范（writing-guide/publishing）：文章模板、标签体系、发布流程

* SEO 清单（keywords/checklist）：每页必备项、站点地图、结构化数据

* 运维文档（deployment/monitoring/security）：环境变量、CI/CD、日志策略、安全要求

* 变更记录（changelog）：语义化版本与变更类型（Added/Changed/Fixed/Removed）

### 写作与格式规范（执行）

* 标题层级清晰，列表上下保留空行，避免连续空行

* 代码块标注语言（如 `ts`、`css`、`text`），必要时给出简短示例

* Python 文件命名采用 snake_case；文档命名使用短横线或下划线统一

* 公式使用 `$...$` 或 `\[...\]`，避免图片公式

### 维护流程与版本策略

* 采用语义化版本（SemVer）：`vMAJOR.MINOR.PATCH`

* 每次发布更新 `changelog.md` 与本方案文件的版本与修订日期

* GitHub 仓库（https://github.com/qingxuandaoming/）建议启用 Issue/PR 模板与自动化检查（Lint/Typecheck）
